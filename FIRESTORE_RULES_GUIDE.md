# Firestore 보안 규칙 적용 가이드

`firestore.rules`를 적용하면 지금처럼 "인터넷의 아무나" DB에 쓸 수 있는 상태를 막을 수 있습니다.
웹앱은 **익명 로그인(Anonymous Auth)** 후에만 읽고 쓰도록 코드가 이미 준비되어 있고,
걸음 수 Worker는 서비스 계정을 쓰므로 규칙의 영향을 받지 않습니다.

---

## ⚠️ 순서가 중요합니다 (이 순서 그대로!)

익명 로그인을 켜기 **전에** 규칙부터 배포하면, 기존 사용자들의 앱이 전부 "권한 없음" 에러로 멈춥니다.
반드시 아래 순서로 진행하세요.

1. **① 익명 인증 켜기** (Firebase 콘솔)
2. **② 앱 배포** — 익명 로그인 코드가 포함된 새 버전을 GitHub Pages에 push
3. **③ 규칙 배포** — 그다음에 firestore.rules 게시

②와 ③ 사이에 하루쯤 두면 친구들 브라우저/PWA 캐시가 새 코드로 갈아타서 더 안전합니다.

---

## ① 익명 인증 켜기 (필수)

1. https://console.firebase.google.com → 프로젝트 **mygraph-6e4c8**
2. 왼쪽 메뉴 **빌드(Build) → Authentication**
3. 상단 **Sign-in method(로그인 방법)** 탭
4. 제공업체 목록에서 **익명(Anonymous)** 클릭 → **사용 설정** 토글 켜기 → **저장**

기존 이메일/비밀번호(관리자용) 제공업체는 그대로 두면 됩니다.

## ② 새 앱 코드 배포

익명 로그인 코드가 들어간 버전을 GitHub Pages에 push 하고,
본인 폰에서 페이지를 한 번 열어 정상 동작(그래프 로딩)을 확인합니다.

## ③ 규칙 배포

1. Firebase 콘솔 → **빌드 → Firestore Database**
2. 상단 **규칙(Rules)** 탭
3. 편집기 내용을 전부 지우고, 저장소의 `firestore.rules` 파일 내용을 통째로 붙여넣기
4. **게시(Publish)** 클릭 (반영까지 최대 1분 정도)
5. 확인:
   - 앱(로그인 상태)에서 체중 입력 → 정상 저장되어야 함
   - 시크릿 창에서 콘솔 열고 아무 인증 없이 REST로 접근하면 `PERMISSION_DENIED`가 떠야 함

---

## 이 규칙이 막아주는 것 / 못 막는 것 (솔직한 한계)

**막아주는 것**
- 앱 없이 프로젝트 ID만 알아낸 외부인이 데이터를 읽거나 쓰는 것 (drive-by)
- 봇/크롤러의 무작위 쓰기
- 이상한 값 저장 (체중 500kg 초과, 걸음 수 20만 초과, 이름 20자 초과 등)
- 관리자가 아닌 사람의 사용자 삭제, settings 변경

**못 막는 것**
- 앱을 열 수 있는 사람(= 친구들)은 익명 로그인이 자동으로 되므로, 마음먹으면
  개발자 도구로 **다른 친구의 기록도** 수정할 수 있습니다.
- 즉 이건 "친구 그룹 수준" 보안입니다. 서로 신뢰하는 소규모 그룹에는 충분하지만,
  모르는 사람에게 URL을 공개하는 서비스 수준은 아닙니다.
- 개인별 계정 분리(진짜 로그인)를 도입하기 전까지는 이 한계를 감수하는 설계입니다.

## 롤백 방법 (문제 생겼을 때)

앱이 갑자기 안 되면 규칙 때문일 가능성이 높습니다. 두 가지 방법:

**방법 A — 이전 버전으로 되돌리기 (권장)**
1. Firestore → 규칙 탭에서 편집기 왼쪽의 **버전 기록** 목록 확인
2. 이전 버전 선택 → 내용 확인 → 다시 **게시**

**방법 B — 임시로 전부 열기 (응급용, 오래 두지 말 것)**

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

붙여넣고 게시하면 규칙 적용 전과 같은 상태(누구나 읽기/쓰기)로 돌아갑니다.
원인 파악 후 반드시 다시 잠그세요.
